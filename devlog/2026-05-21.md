## 2. 剪贴板重复记录修复

**问题**: 用户在 PasteEcho 中点击 Copy/复制后，`pasteItem()` 写入 `NSPasteboard.general` 触发 `changeCount` 变化，0.5s 后 `ClipboardMonitor.poll()` 检测到变化，将相同内容再次加入列表。

**根因**: 将自己的写入行为当作外部复制事件处理。

**修复**: `Sources/Services/DataStore.swift:46` — `add()` 方法添加 contentHash 去重：

```swift
func add(_ item: ClipboardItem) {
    if let hash = item.contentHash,
       let existingIdx = items.firstIndex(where: { $0.contentHash == hash }) {
        items.remove(at: existingIdx)
    }
    items.insert(item, at: 0)
    // ...
}
```

逻辑：
- 新 item 的 contentHash 与已有 item 相同时，移除旧条目再插入新条目（带新 timestamp）
- 重复内容自动浮现到顶部，而非创建重复记录
- 同时处理自复制和外部重复复制两种场景

## 3. 窗口模式切换

**新增功能**: 支持两种窗口模式，可在设置页实时切换。

### 数据模型

`Sources/Models/Settings.swift` — 新增 `WindowMode` 枚举：

```swift
enum WindowMode: String, Codable, CaseIterable, Identifiable, Sendable {
    case topDocked   // 顶部吸附模式（默认）
    case freeWindow  // 自由窗口模式
}
```

`Settings` 新增 `var windowMode: WindowMode = .topDocked`

### 设置页

`Sources/Views/SettingsView.swift` — 新增 Window Mode radio group picker，窗口高度 200→280。保存时同步写入 `settings.json`。

### AppDelegate 双模式

| 模式 | 实现 | 行为 |
|------|------|------|
| topDocked | `NSPopover` + `.transient` | 吸附菜单栏，点击外部自动关闭 |
| freeWindow | `NSWindow` + `.titled .closable .resizable` | 可拖动、可缩放、可长期停留桌面 |

- `togglePopover()` 读取 `dataStore.settings.windowMode` 分发到对应实现
- `toggleDockedPopover()` — 原 NSPopover 逻辑
- `toggleFreeWindow()` / `showFreeWindow()` — NSWindow 生命周期管理
- `isReleasedWhenClosed = false` — 关闭按钮隐藏窗口而非销毁，再次点击菜单栏图标恢复
- 热键 Cmd+Shift+V 在两种模式下均正常工作

### 编译验证

|-----|--------|-------|------|------|
| macOS 13 | exit 0 | exit 0 | 0 | 0 |

## 4. Detail 窗口独立化

**问题**: Detail 在 popover 内展示，空间有限，不适合长文本/长代码阅读。

**修复**: 整套改为独立 NSWindow

| 文件 | 修改 |
|------|------|
| `Sources/Views/DetailContentView.swift` | **新建** — 独立 detail 视图，等宽字体文本、ScrollView、时间戳 |
| `Sources/Views/ClipboardActionPopover.swift` | "Detail" 按钮改为调用 `AppDelegate.showDetailWindow(for:)`，移除内联 detailView |
| `Sources/App/AppDelegate.swift` | 新增 `showDetailWindow(for:)` — 创建独立的 NSWindow |

Detail 窗口特性：
- 默认 600×450，适配长文本阅读
- `styleMask: [.titled, .closable, .resizable]` — 支持拖动缩放
- `minSize: 320×200` — 防止缩得过小
- `setFrameAutosaveName("PasteEchoDetailWindow")` — 系统原生窗口尺寸/位置记忆
- `NSWindowDelegate.windowWillClose` — 补充 UserDefaults 保存 frame
- 文本使用 `.monospaced` 字体，适合阅读代码
- 再次点击同一 item 的 Detail，先关闭旧窗口再创建新窗口

## 5. 自由窗口模式优化

### 5.1 删除无用 + 按钮

`Sources/App/AppDelegate.swift:147` — `showFreeWindow()` 中：
```swift
window.standardWindowButton(.zoomButton)?.isHidden = true
```
绿色最大化按钮（zoom button）在该模式下无意义，隐藏后标题栏只保留关闭按钮。

### 5.2 窗口置顶功能（全局常驻）

| 文件 | 修改 |
|------|------|
| `Sources/ViewModels/AppViewModel.swift` | 新增 `isWindowPinned`、`onWindowPinToggled` 回调、`toggleWindowPin()` |
| `Sources/Views/PopoverRootView.swift` | header bar 常驻 pin 按钮（移除 `if isFreeWindowMode` 条件） |
| `Sources/App/AppDelegate.swift` | `setupWindowPinCallback()` 根据模式切换 `window.level` 或 `popover.behavior` |

两种模式下的 pin 行为：

| 模式 | 未置顶 | 置顶 |
|------|--------|------|
| freeWindow | `window.level = .normal` | `window.level = .floating` |
| topDocked | `popover.behavior = .transient` | `popover.behavior = .applicationDefined` |

- pin 图标始终显示在 header bar
- 激活时橙色 `.pin.fill`
- topDocked 模式下置顶 = popover 不随点击外部关闭

### 5.3 窗口模式实时切换

**问题**: 在设置页切换模式后 UI 无反应，需重新点击菜单栏图标。

**修复**: 新增 `onWindowModeChanged` 回调链，设置变更后立即生效。

| 文件 | 修改 |
|------|------|
| `Sources/Services/DataStore.swift` | 新增 `onWindowModeChanged: (@MainActor () -> Void)?` |
| `Sources/Views/SettingsView.swift` | windowMode 的 onChangeCompat 中额外调用 `onWindowModeChanged?()` |
| `Sources/App/AppDelegate.swift` | `setupWindowModeObserver()` 设置回调 |

切换逻辑：
- **→ 自由窗口**: 关闭 popover → 立即 `showFreeWindow()`，自动聚焦
- **→ 顶部吸附**: 隐藏 clipboardWindow → 立即 `toggleDockedPopover()`，吸附菜单栏

无需重新点击菜单栏图标，实时生效。

## 6. 导出完整 .app

当前环境无 Xcode.app（仅 Command Line Tools），`xcodebuild` 不可用。采用手动构建：

```bash
# 1. swiftc 编译 arm64 binary (strict concurrency, -O)
# 2. iconutil 从 PNG 生成 AppIcon.icns
# 3. 创建 Info.plist (LSUIElement=YES)
# 4. codesign --force --sign - (ad-hoc)
# 5. zip 打包
```

| 项目 | 值 |
|------|-----|

| 签名 | ad-hoc |
