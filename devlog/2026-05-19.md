### Bug 1: 粘贴板不实时刷新 ✅

**根因**: `AppViewModel.displayItems` 是计算属性（computed property），内部读取 `dataStore.items`（`@Published`），但 `AppViewModel` 未订阅 `DataStore.$items`。SwiftUI 无法感知 `dataStore.items` 变化，导致 `displayItems` 不触发重新渲染。

**修复**: `Sources/ViewModels/AppViewModel.swift`
- `displayItems` 从计算属性改为 `@Published var`
- `init` 中通过 Combine 订阅 `dataStore.$items`，变化时调用 `updateDisplayItems()`
- 同步订阅 `$searchQuery`，搜索关键词变化时同样刷新
- 添加 `import Combine`

### Bug 2: 启动自动弹出 Settings ✅

**根因**: `PasteEchoApp.swift` 只声明了一个 `Window("Settings", id: "settings")` scene，没有主 `WindowGroup`。macOS 在首次启动时自动打开该窗口（窗口状态恢复机制）。

**修复**: 3 个文件
- `Sources/App/PasteEchoApp.swift`: 移除 `Window` scene，改为 `WindowGroup { EmptyView() }`（不创建可见窗口）
- `Sources/App/AppDelegate.swift`: 新增 `showSettings()` 方法，手动创建 `NSWindow` + `NSHostingView(SettingsView)`，`isReleasedWhenClosed = false` 防止关闭后释放
- `Sources/Views/PopoverRootView.swift`: `openSettings()` 改为调用 `(NSApp.delegate as? AppDelegate)?.showSettings()`

### Bug 3: 点击粘贴内容交互 ✅

**问题**: 点击卡片直接复制，无操作菜单。

**修复**: 2 个文件
- **新建** `Sources/Views/ClipboardActionPopover.swift`: 点击卡片弹出的操作面板
  - 默认显示 4 个操作选项：Detail / Copy / Pin(Unpin) / Delete
  - Detail 进入详情页：完整内容（长文本可滚动）、精确时间戳、返回按钮
  - Copy 重新写入系统粘贴板，带 checkmark 反馈
  - Pin/Unpin 切换置顶状态（持久化于 items.json）
  - Delete 删除记录，UI 实时刷新
- **修改** `Sources/Views/ClipboardCard.swift`:
  - `.onTapGesture` 改为弹出 `ClipboardActionPopover`（`.popover`）
  - 移除悬停时显示的 pin/delete 按钮（操作统一到点击弹窗）

## Phase 4: Bug 深度修复 (根因分析)

### Bug 2 (再次): 启动空白窗口 ✅ (关键修复)

**上一次修复的问题**: Phase 3 将 `Window("Settings", id:)` 替换为 `WindowGroup { EmptyView() }`，但这仍然创建了窗口！

**根因**: `LSUIElement = YES` 仅隐藏 Dock 图标和 App Switcher，**不隐藏窗口**。`WindowGroup` 始终创建一个可见窗口，`EmptyView()` 内容导致窗口显示为空白。

**关键修复 (3 文件)**:
- **删除** `Sources/App/PasteEchoApp.swift` (SwiftUI `@main App` + `WindowGroup`)
- **新建** `Sources/main.swift`: 传统 AppKit 启动方式，`MainActor.assumeIsolated` 确保 Swift 6 actor 隔离兼容
- `Sources/App/AppDelegate.swift`: 移除对 `@NSApplicationDelegateAdaptor` 的隐式依赖，改为 `main.swift` 显式设置 `app.delegate = AppDelegate()`

```swift
// main.swift
import AppKit
MainActor.assumeIsolated {
    let app = NSApplication.shared
    let delegate = AppDelegate()
    app.delegate = delegate
    app.run()
}
```

## Phase 5: 剪贴板监听深度重构 (Combine → @MainActor Callback)

### 问题

尽管已修复 SwiftUI App 生命周期问题，剪贴板仍然不实时刷新。经过逐环节排查，确认数据能被持久化（重启后可见），说明 **DataStore.add() 正常，但 UI 未收到变化通知**。

### 根因

Combine 跨对象订阅 (`PassthroughSubject` + `$items.sink`) 在 Swift 6 strict concurrency 下有不可靠的边界情况：`@Published` 的 publisher 调度时机、`[weak self]` 与 actor 隔离的交互、Combine 内部线程调度都可能静默丢弃事件。

### 修复: 全面替换为 @MainActor 直接回调

**4 文件重构，消除所有跨对象 Combine**：

| 文件 | 修改前 | 修改后 |
|------|--------|--------|
| [ClipboardMonitor.swift](Sources/Services/ClipboardMonitor.swift) | `Timer.publish` + `PassthroughSubject` | `Timer.scheduledTimer` + `@MainActor` callback |
| [DataStore.swift](Sources/Services/DataStore.swift) | 无回调 | `onItemsChanged: (@MainActor () -> Void)?`，`add()` / `remove()` 触发 |
| [AppViewModel.swift](Sources/ViewModels/AppViewModel.swift) | `dataStore.$items.sink` | `dataStore.onItemsChanged = { ... }` 直接回调 |
| [AppDelegate.swift](Sources/App/AppDelegate.swift) | `newCapturePublisher.sink` + `cancellables` + `import Combine` | `clipboardMonitor.onCapture = { ... }` 直接回调 |

### 数据流 (最终版)

```
NSPasteboard.changeCount 变化
  → Timer.scheduledTimer(0.5s) → MainActor.assumeIsolated { poll() }
  → onCapture?(clipboardCapture)              // @MainActor 直接回调
  → AppDelegate: dataStore.add(item)
  → DataStore: items.insert() + onItemsChanged?()  // @MainActor 直接回调
  → AppViewModel: updateDisplayItems()
  → displayItems = sorted                     // @Published 触发 SwiftUI 重绘
```

每个环节都是 `@MainActor` 同步调用，无 Combine、无调度延迟、无 weak 引用丢失。

### 附: togglePin 同步刷新

`AppViewModel.togglePin()` 现在直接调用 `updateDisplayItems()`，不再依赖 Combine 订阅。
